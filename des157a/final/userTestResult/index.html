User Test Observations and Results

What did we do?

For today's class, we had to test our projects by letting users blindly test the pages themselves as if they had stumbled upon the website on their own. We had to give them a set of goals to achieve while testing, and give us feedback accordingly. The feedback session was really informative and helpful in letting us know what works and what we need to work on. This page summarizes the feedback received from three users testing it and what changes I will incorporate into my final project.

Feedback

There was a lot of constructive feedback given to me after the test was completed. Most of it I received from all three users was pretty similar and can be broadly classified as follows -

  • Unclear/Hard-to-understand Instructions - During the testing,I had observed the users taking a bit of time to understand the instructions. A common feedback was that the instructions at the beginning of the game were a bit too wordy and hard to understand. The instructions did not give them a clear enough idea of what was going to come up and that it took them a while to truly understand what I was trying to convey.
  • Quit/Restart Game - Once the game starts, the user has no way to exit the quit to the end to check their score report or just directly restart the game. The general opinion was it would be a good option to give a user an exit option so that they do not have to wait out the entire 30 seconds of the game before they can either restart or quit the game.
  • Feedback on Score Report - The testers mentioned that they would have liked to see some visual feedback on the final page. It would be based on either their score or accuracy to know whether their performance was good or not, or at least print a message telling the user 'Good Job!' or to 'Try Harder Next Time!'.
  • Font matching - Another suggestion I received was to change the instruction font style to match the retro-pixelated game look.
  • Partially Unclear Game Mechanics - Since the game is randomized, sometimes the same word ('left' or 'right') may appear on the same tile in the next level as well, which can confuse the user as to whether the game is working or has been stalled. The suggestion was that I can change the game such that this confusion is avoided by either making sure the same combination does not repeat or having a better visual indicator of a level being passed.
  • Difficult to judge time - A tester's feedback was that it was difficult to keep track of the time and realize that time is running low since there is no audio or visual feedback of time about to reach to zero. The suggestion was made to change the color of the time so that the user doesn't have to read the time but based on the color can guess how much time is left, which will let them focus on the game more.

Changes I will Make

The process of blind testing was really useful to let us know how a first-time user approaches our website. Taking the feedback and my own observations of the user testing process into account, these are the changes that I will definitely be making -

  • Clearer and Simpler Instructions - Based on user feedback above, I will simplify the instructions language and also try to show it in a bullet point format for greater ease in understanding. I will also try and give them a visual representation of what is coming up by including a small doodle image of the game page and showing a correct example.
  • Quit/Restart - I will add both a quit and restart button in the game page itself. Instead of waiting 30 seconds, the user will be able to either quit to the end screen or restart the game. This will probably be in the top left corner, away from other buttons to prevent the user accidentally clicking it.
  • Score Report Feedback - I plan on adding visual feedback to the score report page based on user performance in the game. This will be done by changing the background color and adding a text depending on how well the user performance is. For example, a gold background for a really good score and a message saying "Good Job!".
  • Font Matching - I will be changing the instruction text and score report text to match the retro byte game theme of the title and buttons.
  • Time Feedback - I will use an animation on the timer css to change it from a white color to a red color which will allow the users to know that time is running out.
